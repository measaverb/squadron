# Squadron — 합성 데이터셋

**Squadron** 엔지니어링 리소스 매니저 프로토타입을 위한 완전 합성 데이터셋.
`generate_data.py`로 시드 `42`를 사용해 생성함; 동일한 시드는 이 데이터셋을
정확히 재현함.

## 출처 및 소스
모든 레코드는 프로그래밍 방식으로 생성됨 — 실제 개인정보나 독점 데이터는
사용하지 않음. 그럼에도 생성기의 *매개변수*는 실제 오픈 데이터에 근거함:

- **스킬/기술 인기도**와 **연차 분포**는 Stack Overflow 연례 개발자 설문조사를
  기반으로 모델링함 — https://survey.stackoverflow.co/
- **작업 제목과 공수 추정치**는 Apache Software Foundation Jira 같은 공개 이슈
  트래커의 관례를 따름 — https://issues.apache.org/

## 파일
- `projects.json` — 프로젝트 3개 (`id`, `name`, `description`, `deadline`)
- `sprints.json` — 스프린트 12개 (`id`, `project`, 스토리 포인트 단위 `capacity`)
- `developers.json` — 개발자 특성 프로필 55개
- `developers.csv` — 위와 동일, 표 형식
- `tasks.json` — 작업 153개; 의존성 그래프는 **DAG** (항상 비순환)
- `tasks_broken.json` — 동일하되, 순환 1개를 주입함 (순환 탐지 데모용)

## 필드 사전
**developer**: `id`, `name`, `level` (1 주니어 – 5 프린시펄), `skills` (목록),
`capacity` (스프린트당 스토리 포인트), `current_load` (이미 확정된 포인트),
`familiar_projects` (프로젝트 id 목록).

**task**: `id`, `project`, `title`, `description`, `tags` (목록), `required_skills`
(목록), `difficulty` (1–5), `estimate` (스토리 포인트), `value` (비즈니스 가치
10–100), `priority` (1 최상 – 4 최하), `depends_on` (선행 작업 id 목록),
`sprint` (스프린트 id), `assignee` (`null`; 할당기가 채움).

## 의도적으로 설계한 시연 사례
각 알고리즘이 보고서와 영상에서 보여줄 교육적 사례를 갖도록 의도적으로 심어둠.

1. **임계 경로 (DAG에서 최장 경로).** 무거운 의존성 체인이 첫 P1 작업 8개를
   관통함: `T0001, T0002, T0003, T0004, T0005, T0006, T0007, T0008` (추정치 13/8 교대,
   총 84포인트의 중심 골격). 임계 경로는 이들을 곧장 관통해야 함.

2. **스프린트 플래너 반례 (0/1 배낭 문제).** 스프린트 `P3-S4`는 용량 `10`이며
   항목 `T0151, T0152, T0153` = A(6,60), B(5,40), C(5,40)을 포함. 가치 밀도 기준
   그리디는 A(가치 60)를 택하고 나머지는 넣지 못함; 최적해는 B+C(가치 80).
   그리디/최적 간 격차를 드러냄.

3. **할당기 희소성 (헝가리안 대 그리디).** 스킬 4개가 희소함 — 각각 개발자
   3명만 보유 — 그럼에도 여러 작업이 요구함:
  - `rust`: D035, D047, D016 보유; T0091, T0069, T0102 요구
  - `spark`: D011, D030, D025 보유; T0011, T0048, T0102 요구
  - `terraform`: D018, D041, D045 보유; T0109, T0123, T0032 요구
  - `flutter`: D036, D015, D044 보유; T0095, T0065, T0070 요구
   희소 스킬 보유자는 일반 작업에도 매력적이므로, 그리디(우선순위 정렬) 할당은
   그들을 굳이 필요로 하지 않는 작업에 소진해 희소 스킬 작업을 인력 미배치
   상태로 남길 수 있음; 헝가리안 알고리즘은 할당을 전역적으로 최적화해 이를
   방지함.

## 재현성
`python generate_data.py --seed 42`는 이 데이터셋을 바이트 단위로 동일하게
재생성함. 외부 라이브러리는 필요 없음 (Python 3.8+ 표준 라이브러리만으로 충분).
